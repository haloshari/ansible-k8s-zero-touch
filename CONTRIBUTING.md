# Contributing to Ansible K8s Zero-Touch

First off, thank you for considering contributing to this project! It's people like you that make open-source tools great.

## How can I contribute?

### 1. Reporting Bugs
If you find a bug in the Ansible playbooks or roles, please open an issue on GitHub. Include:
* Your operating system and Ansible version.
* The exact error message from your terminal.
* Steps to reproduce the issue.

### 2. Suggesting Enhancements
Want to add support for a new CNI (like Cilium instead of Calico) or support for a new Linux distribution? Open an issue detailing your proposal so we can discuss it!

### 3. Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added new Ansible variables, make sure to document them.
3. Ensure your code maintains the "Zero-Touch" philosophy (no manual intervention required).
4. Issue that pull request!

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Treat everyone with respect.
